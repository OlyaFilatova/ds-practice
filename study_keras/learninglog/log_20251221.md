- [x] https://keras.io/getting_started/intro_to_keras_for_engineers/
    - Keras 3 is a deep learning framework that works with TensorFlow, JAX, and PyTorch interchangeably
    - model-building options that Keras offers include:
        - The Sequential API
        - The Functional API
        - Writing your own models yourself via subclassing (for advanced use cases)
    - enables to write custom Layers, Models, Metrics, Losses, and Optimizers
    - models can be trained and evaluated on a wide variety of data sources
        - NumPy arrays
        - Pandas dataframes
        - TensorFlow tf.data.Dataset objects
        - PyTorch DataLoader objects
        - Keras PyDataset objects
- [x] https://keras.io/getting_started/about/
    - The core data structures of Keras are layers and models
- [x] https://keras.io/keras_3/
    - ability to dynamically select the backend that will deliver the best performance for the model without having to change anything in the code
    - Any Keras 3 model can be instantiated as a PyTorch Module, can be exported as a TensorFlow SavedModel, or can be instantiated as a stateless JAX function
- [x] https://keras.io/getting_started/ecosystem/
    - KerasHub is a natural language processing library
    - KerasTuner is a scalable hyperparameter optimization framework
        - comes with Bayesian Optimization, Hyperband, and Random Search algorithms built-in, and is designed to be easy to extend in order to experiment with new search algorithms
    - AutoKeras provides high-level end-to-end APIs to solve machine learning problems in a few lines
    - BayesFlow - library for amortized Bayesian workflows using generative neural networks
